## 本轮需求书调整

### `docs/phase1/requirements-supplement.md`

主要修正：

1. 将原来的“3 个补充模块”改为 4 个补充模块。
2. 明确 v0.4 场景 7 的“筛选基金”和“保存为自选候选池”是两个动作。

6. 明确 `diff_research_packets` 至少覆盖经理、规模、净值指标、公开持仓、风格暴露、风险提示、证据和结论状态。

### `docs/phase2/requirements.md`

主要修正：

1. 将二期拆成 P2A/P2B/P2C 三个可验收小闭环。
2. P2A 聚焦 ORM + 迁移 + 实验管理 + 前端 Phase 1 API 页面 scaffold。
3. P2B 聚焦模拟持仓、动态归因、评分的算法验证和回测。
4. P2C 才做受控产品化视图。
5. 明确未通过回测和阈值的估计结果只能保留在实验结果与 `estimated_*` 字段中，不得进入默认高置信度结论、默认评分或排名。
6. P2A 第一轮完成条件新增：`ruff`、`pytest`、`npm run build`、`check-data` 全过，并有实验管理测试覆盖。
