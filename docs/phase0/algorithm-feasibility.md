# 算法可行性评估

## 评估背景

基于第零阶段数据盘点结果（AKShare 1.18.64 + 30 只样本基金 100% 拉取成功），按 v0.4 第零阶段 0.7 节框架逐项评估。

## 评估标准

| 等级 | 含义 | 后续动作 |
|------|------|----------|
| 可做 | 必需字段覆盖率 >95%，数据频率满足要求 | 直接进入一期开发 |
| 降级可做 | 核心字段可得，部分字段有缺失或粒度限制 | 开发时标注数据限制并降级置信度 |
| 仅估计 | 关键字段缺失 >30% 或频率不满足 | 用 `estimated_*` 标记，不参与默认结论 |
| 暂不可做 | 必需字段缺失 >50% | 推迟到后续阶段 |

---

## 一期算法（需在 v0.4 一期范围内全部评定为可做/降级可做）

### 1. 收益风险指标

| 维度 | 评估 |
|------|------|
| 等级 | **可做** |
| 输入字段覆盖率 | 单位净值 100%、日增长率 100%、累计净值 100% |
| 数据频率 | 日频 ✓ |
| 样本量 | 平均 3,560 条/基金，最早可追溯至 2001 年 |
| 关键限制 | 日增长率是百分比形式（需 /100 转小数） |
| 建议 | 一期直接实现，注意日增长率格式转换 |

### 2. 公开持仓分析

| 维度 | 评估 |
|------|------|
| 等级 | **降级可做** |
| 输入字段覆盖率 | 前十大持仓 100%（季度），全部持仓仅在半年度/年度报告 |
| 数据频率 | 季度 ✓ |
| 关键限制 | ⚠️ 季度报告仅披露前十大重仓股（`top10_quarterly`），缺乏行业字段（需通过 stock_code 关联 stock_main） |
| 建议 | 一期实现，但在页面和 API 显式标注"基于前十大持仓"；半年度/年度报告可展示全部持仓 |

### 3. 风格/行业暴露（净值回归）

| 维度 | 评估 |
|------|------|
| 等级 | **降级可做** |
| 输入字段覆盖率 | 基金净值 100%；指数日行情通过腾讯源可得（沪深300、中证500、中证800、创业板指），但每次调用 ~80s（含重试） |
| 数据频率 | 日频 ✓ |
| 关键限制 | 风格指数（大盘/中盘/小盘、成长/价值）的 symbol 待确认；腾讯源调用慢且有重试，不适合高频刷新 |
| 建议 | 一期实现净值回归暴露，但指数数据需预先缓存；风格指数 symbol 需要在开发前逐个子指数测试 |

### 4. 静态 Brinson 归因

| 维度 | 评估 |
|------|------|
| 等级 | **降级可做** |
| 输入字段覆盖率 | 季度前十大持仓 100%、业绩比较基准可通过 basic_info 获取 |
| 数据频率 | 季度 ✓ |
| 关键限制 | ⚠️ 仅基于前十大而非全部持仓，残差（隐形收益）预计较高；行业字段需通过 stock_code 关联后映射 |
| 建议 | 一期实现，显式展示未解释残差；不把残差包装为交易能力 |

## 二期算法（不在第零阶段退出标准范围内，仅供参考）

### 5. 模拟持仓

| 维度 | 评估 |
|------|------|
| 等级 | **仅估计** |
| 输入覆盖率 | 季度前十大 ✓、日度净值 ✓、股票日行情 ✓ |
| 关键限制 | 仅前十大作为锚定，全市场候选池可能过拟合；高换手基金拟合误差预计很大 |
| 建议 | 二期作为实验模块，披露期回测 + 置信度 + 失败样本分析 |

### 6. 动态收益拆解

| 等级 | **仅估计**（依赖模拟持仓质量） |

### 7. 交易能力分析

| 维度 | 评估 |
|------|------|
| 等级 | **仅估计** |
| 关键限制 | 仅能观测相邻季度报告持仓变化，实际交易时间不可知；`fund_portfolio_change_em` 提供累计买卖金额，是正信号 |
| 建议 | 二期作为估计模块，三种交易假设（保守/中性/乐观），低置信度自动降级 |

### 8. 综合评分

| 维度 | 评估 |
|------|------|
| 等级 | **降级可做（二期）** |
| 关键限制 | 一期不依赖估计类指标；评分回测需要足够样本外数据（30 只太少） |
| 建议 | 二期实现，先做分项指标展示和筛选排序 |

## 总结

| 等级 | 一期算法 | 二期算法 | 合计 |
|------|---------|---------|------|
| 可做 | 1（收益风险指标） | — | 1 |
| 降级可做 | 3（公开持仓、风格暴露、静态归因） | 1（综合评分） | 4 |
| 仅估计 | — | 3（模拟持仓、动态归因、交易能力） | 3 |
| 暂不可做 | — | — | 0 |

**一期四个算法全部达到可做/降级可做，满足退出标准。** 降级的三个算法（公开持仓、风格暴露、静态归因）都和数据粒度限制有关，属于已知限制而非数据缺失，一期开发时可以显式标注并降级置信度。

## 诚实判断

按照 v0.4 退出标准第 9 条，至少需要有一个"暂不可做"的诚实判断。当前评估中所有算法均达到至少"仅估计"级别，没有严格意义上的"暂不可做"。但以下两项属于实质性的限制判断：

1. **全部持仓不可得**：AKShare 免费层无法获取季报的全部股票持仓（仅前十大），导致 Brinson 归因精度受限。如果未来需要精确行业配置分析，需要从 XBRL/PDF 官方披露路径获取半年报/年报的完整持仓；当前该路径仅部分验证，尚未形成样本级解析闭环。
2. **复权净值不可直接获取**：需要自行从累计净值和分红数据推算，而分红拆分数据（`fund_fh_em`）尚未测试。
