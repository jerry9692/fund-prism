## 1. 证监会 XBRL 专区（持仓数据）

### 基本信息

- **网址**: `http://eid.csrc.gov.cn/fund/disclose/stat_fund_view.html`
- **XBRL 分类标准文档**: `http://eid.csrc.gov.cn/fund-xbrl/TaxonomyFrameworkOverview.htm`（最后更新 2026-03-09）
- **背景**: 2009 年上线，证监会强制要求所有公募基金以 XBRL 格式报送季度/半年度/年度报告
- **审计状态**: 官方平台和基金 XBRL 信息披露制度存在可确认；当前仓库没有保留样本基金 XBRL 文件下载、解析日志或字段覆盖率，因此不得标记为“自动化已验证”。

### XBRL 模板覆盖的数据

根据证监会公告，XBRL 报送涵盖以下模板：

| 模板编号 | 内容 | 公告文号 |
|----------|------|----------|
| 第 1 号 | 季度报告 | [2008]36 号 |
| 第 2 号 | 净值公告 | [2012]5 号 |
| 第 3 号 | 年度报告和半年度报告 | [2010]5 号 |
| 第 4 号 | 临时公告 | [2010]32 号 |

### 手动验证步骤

1. 打开 `http://eid.csrc.gov.cn/fund/disclose/stat_fund_view.html`
2. 搜索一只样本基金（如 000001 华夏成长混合）
3. 查看是否有：
   - 基金基本信息
   - 定期报告列表（可下载 XBRL/XML 格式）
   - 持仓明细数据
4. 下载一份 XBRL 文件，检查：
   - 是否可以用 Python 的 `xml.etree` 或 `lxml` 解析
   - 是否包含股票持仓明细（代码、名称、比例、市值）
   - 是否包含债券持仓
   - 行业分类字段是否标准
5. 记录以下信息：
   - 页面是否有反爬机制
   - 单个基金有多少份 XBRL 文件
   - 文件命名规则（是否可程序化构造 URL）
   - XBRL 文件大小

### 当前结论

XBRL 平台是最理想的 A 级数据源方向，但第零阶段当前只完成到“路径存在/平台级可达性”：
- **优点**: 官方权威、覆盖全（净值+持仓+经理）、结构化格式（无需 PDF 解析）
- **缺点**: 可能存在访问限制、API 不公开、历史数据范围待确认
- **一期使用约束**: 不能作为默认主数据源；若 API 返回 `source_level: A`，必须绑定具体 XBRL/PDF 文件、下载时间和解析证据。

## 2. 巨潮资讯（公告 PDF）

### 基本信息

- **网址**: `http://www.cninfo.com.cn/`
- **背景**: 证监会指定的上市公司和基金信息披露平台，所有基金定期报告必须在此发布
- **数据格式**: PDF（定期报告全文）
- **审计状态**: 首页和基金公告入口可达；样本 PDF 已下载到 gitignored 缓存，仓库保留 URL、hash、页数和关键词解析摘要。

### 手动验证步骤

1. 打开 `http://www.cninfo.com.cn/`
2. 在"基金"分类下搜索基金代码 000001
3. 查看是否有：
   - 定期报告 PDF 列表（季度/半年度/年度）
   - 招募说明书 PDF
   - 基金合同 PDF
4. 下载一份季报 PDF 和一份年报 PDF
5. 检查 PDF 内容：
   - 季报是否包含前十大重仓股表格
   - 年报/半年报是否包含全部持仓
   - 表格是否能用 `pdfplumber` 或 `camelot` 提取
   - 报告格式是否标准化

### 当前结论

巨潮资讯是可靠的 A 级公告入口方向，第零阶段已完成最小证据闭环：
- **优点**: 官方指定、覆盖全、PDF 格式标准化
- **缺点**: PDF 表格结构化抽取仍需后续验证，自动化难度高
- **一期使用约束**: 可以作为 Evidence 的 A 级引用；结构化持仓仍以 AKShare/B 级缓存或未来 XBRL/PDF 表格解析为主。

## 4. 自动化难度评估

| 路径 | 自动化难度 | 理由 |
|------|-----------|------|
| 证监会 XBRL | **中** | XML 格式天然可解析，但可能需要申请访问权限 |
| 巨潮资讯 PDF | **高** | PDF 表格提取准确率有限，且每个报告单独下载 |
| 基金公司官网 | **中-高** | 各家公司页面结构不同，需要逐个适配；可能存在反爬 |
| 交易所公告 | **中** | 上交所/深交所公告页面结构相对稳定 |

## 5. 推荐优先级

1. **首选**: 证监会 XBRL 专区 — 如果可用，这是最理想的 A 级数据源
2. **备选**: 巨潮资讯 — 作为 PDF 公告的可靠来源，用于证据引用和人工复核
3. **补充**: 基金公司官网 — 各公司独立维护，不适合作为统一数据入口，但可作为特定基金的补充验证

## 6. 后续建议

- 第零阶段只需**验证可达性和基本可行性**，不需要实现完整的自动化爬取
- 如果 XBRL 平台可用，一期优先做“一只样本基金 XBRL 下载 + Arelle 解析 + 字段覆盖记录”的最小闭环，再升级为 A 级结构化源
- 一期可以先把巨潮 PDF 作为 A 级 Evidence 引用来源
- 一期接入 A 级结构化持仓前至少完成以下验证：
  - [ ] 访问 XBRL 专区，确认是否可以查看特定基金的持仓数据
  - [ ] 从巨潮资讯下载 1 份季报 PDF，用 pdfplumber/camelot 尝试提取持仓表格
  - [ ] 访问 1 家基金公司官网，确认净值数据是否可以通过非登录方式获取
